	O codigo produzido é um codigo para gerenciamento de substituição de pagina para a disciplina de Sistemas Operacionais,
os algoritmos utilizados foram os algoritmos de fifo(first in first out) e o altoritmo de second chance.
	O maior problema para confecção do codigo foi principalmente o entendimento do funcionamento do simulador base,
como estavam estruturados os Dados de paginas e frames e como estavam representados cada um dos valores da estrutura.

	Para compilação e testes o codigo rodou em ambiente linux, ubuntu 16, para isso é feita a compilação do codigo e
posteriormente ele deve rodar com os seguintes argumentos: <nome do algoritmo> e <clock> em seguida as entradas de dados podem 
ser feita manual ou por exemplo como o do .dat, para isso então por exemplo ./vmm second_chance 10 < anomaly.dat
